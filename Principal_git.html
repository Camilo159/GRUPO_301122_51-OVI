Un control de versiones es un sistema que registra los cambios realizados sobre archivos a través del tiempo, de esta manera puede revisar y retornar a versiones específicas después, así como comparar cambios a través del tiempo y ver quien los ha realizado, permite revertir cambios realizados y retornar a estados anteriores, inclusive puede revertir todos los cambios realizados sobre un proyecto entero.

Existen diferentes tipos de sistemas gestores de versionamiento:

  1. Locales
  2. centralizados
  3. distribuidos

Git es un sistema de control de versiones distribuido, libre y open source diseñado originalmente por Linus Torvalds, creador del Kernel de Linux el 7 de abril de 2005 bajo la licencia GNU GPL v2, está pensado especialmente para manejar proyectos tanto grandes como pequeños de una manera rápida y eficiente.

En un sistema de control de versiones distribuido en lugar de tener un único repositorio central en el cual los clientes se sincronizan como en los sistemas centralizados, la copia local de cada uno es un repositorio completo, de esta manera si un servidor se cae cualquiera de los repositorios de los clientes puede copiarse en el servidor para restaurar el repositorio, ya que los clientes hacen una copia de seguridad completa de todos los datos del repositorio.

Para comenzar a utilizar Git debemos instalarlo primero en nuestro PC, Podemos descargar Git desde su página oficial: git-scm.com

Github es una plataforma de desarrollo online en la cual podemos alamcenar nuestros repositorios utilizando licencias libres y Open source a través del sistema de control de versiones Git de manera gratuita, de esta forma podemos revisar código de otras personas y colaborar en proyectos con diferentes desarrolladores a través del mundo.

Podemos crearnos una cuenta de Github desde su página oficial: Github.com
